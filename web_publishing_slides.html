PART 04
CSS 단위 — 언제 뭘 쓰나요?
px
픽셀 (절대값)
고정 크기가 필요할 때
border · padding
border: 1px; padding: 16px;
%
퍼센트 (상대값)
부모 기준 비율
레이아웃 분할
width: 50%; margin: 5%;
vw
vh
뷰포트 기준
화면 전체 대비
풀스크린 섹션
height: 100vh; width: 50vw;
rem
루트 폰트 기준
반응형 · 접근성
1rem = 보통 16px
font-size: 1.5rem; margin: 2rem;
💡 처음엔 px 위주로, 레이아웃은 %·flex, 반응형은 rem·vw 조합이 실무 기본입니다.
PART 05
Flexbox vs CSS Grid
Flexbox — 1차원 (한 방향)
가로 또는 세로, 한 줄로 배치할 때
.nav { display: flex; gap: 12px; align-items: center; }
메뉴1
메뉴2
메뉴3
항목 A
항목 B
✅ 내비게이션 · 버튼 그룹 · 카드 한 줄
CSS Grid — 2차원 (행 + 열 동시)
행과 열을 동시에 제어할 때
.layout { display: grid; grid-template-columns: 1fr 2fr; gap: 12px; }
헤더
사이드바
본문
푸터
✅ 전체 페이지 레이아웃 · 카드 그리드 · 대시보드
Flex = 한 줄 정렬이 목적
|
Grid = 행·열 동시 배치가 목적
|
같이 써도 됩니다 — Grid로 전체 잡고 Flex로 내부 정렬

PART 07
가상 선택자 (Pseudo Selector)
:pseudo-class — 상태 선택자
선택자 의미 예시
:hover 마우스 올렸을 때 a:hover
:focus 포커스됐을 때 input:focus
:active 클릭 누르는 순간 button:active
:nth-child(n) n번째 자식 li:nth-child(2)
:first-child 첫 번째 자식 p:first-child
:last-child 마지막 자식 li:last-child
:not(선택자) 해당 제외 전부 p:not(.on)
::pseudo-element — 가상 요소
선택자 의미 예시
::before 요소 앞에 내용 삽입 div::before
::after 요소 뒤에 내용 삽입 div::after
::placeholder placeholder 스타일 input::placeholder
::selection 드래그 선택 스타일 ::selection
실전 예시
button:hover { background: var(--blue); } tr:nth-child(odd) { background: #f5f5f5; } .item::before { content: "▶ "; color: var(--blue); }
PART 08
반응형 웹 (Responsive Web)
반응형이란?
화면 크기(PC · 태블릿 · 모바일)에 따라 레이아웃이 자동으로 바뀌는 웹
PC — 3열
태블릿 — 2열
모바일 — 1열
viewport 메타 태그 — 필수!
<meta name="viewport" content="width=device-width, initial-scale=1.0">
이 태그 없으면 모바일에서 PC 화면을 축소해서 보여줌
미디어 쿼리 (Media Query)
화면 너비에 따라 다른 CSS를 적용하는 문법
/* 기본: PC용 3열 */ .grid { display: grid; grid-template-columns: 1fr 1fr 1fr; } /* 태블릿 이하 (768px) */ @media (max-width: 768px) { .grid { grid-template-columns: 1fr 1fr; } } /* 모바일 (480px 이하) */ @media (max-width: 480px) { .grid { grid-template-columns: 1fr; } }
💡 Mobile First — 모바일 기본 작성 후 min-width로 확장하는 방식이 권장됩니다
PART 09
CSS 방법론 — BEM / OOCSS
BEM — Block Element Modifier
클래스 이름만 보고 구조를 파악할 수 있게 하는 네이밍 규칙
/* Block__Element--Modifier */ .card { } /* Block */ .card__title { } /* Element */ .card__button { } /* Element */ .card--featured { } /* Modifier */ .card__button--disabled { }
HTML 사용 예
<div class="card card--featured"> <h2 class="card__title">제목</h2> <button class="card__button">확인</button> </div>
OOCSS — 구조와 스킨 분리
재사용 가능한 CSS를 위해 구조(Structure)와 스킨(Skin)을 분리
/* ❌ 중복 — 안 좋은 방식 */ .btn-blue { padding: 8px 16px; border-radius: 6px; background: blue; } /* ✅ OOCSS — 구조 / 스킨 분리 */ .btn { padding: 8px 16px; border-radius: 6px; } .btn--blue { background: blue; } .btn--red { background: red; }
왜 쓰나요?
✅ 클래스명만 봐도 역할 파악 가능
✅ CSS 재사용으로 코드량 감소
✅ 팀 협업 시 네이밍 충돌 방지
✅ 유지보수 시 수정 범위가 명확
PART 10
유틸리티 클래스 (Utility-First CSS)
하나의 클래스 = 하나의 CSS 속성. 조합해서 UI를 만드는 방식 — Tailwind CSS가 대표적
BEM 방식 vs 유틸리티 방식 비교
❌ BEM 방식 — CSS 파일 따로 작성
/* CSS 파일 */ .card { display: flex; padding: 16px; background: #fff; border-radius: 8px; } /* HTML */ <div class="card">...</div>
✅ 유틸리티 방식 — HTML에서 바로
/* CSS 파일 없음 */ <div class="flex p-4 bg-white rounded-lg shadow"> ...</div>
직접 만들 수도 있어요
자주 쓰는 속성을 단일 클래스로 정의해두면 편리
/* 직접 만드는 유틸리티 */ .flex { display: flex; } .hidden { display: none; } .w-full { width: 100%; } .mt-1 { margin-top: 4px; } .mt-2 { margin-top: 8px; } .text-sm { font-size: 12px; } .bold { font-weight: 700; } .rounded { border-radius: 8px; }
Tailwind CSS 주요 클래스
레이아웃
flex → display:flex grid → display:grid hidden → display:none w-full → width:100% h-screen → height:100vh
간격·크기
p-4 → padding:16px px-4 → padding 좌우 mt-2 → margin-top:8px gap-4 → gap:16px rounded → border-radius
정렬
items-center → align-items:center justify-center → justify-content text-center → text-align:center
색상·텍스트
text-sm → font-size 작게 font-bold → font-weight:700 text-blue-500 → color:blue bg-white → background:#fff
BEM vs 유틸리티 — 언제 뭘?
BEM/OOCSS — 컴포넌트가 복잡하고 재사용이 많을 때, 디자인 시스템 구축 시
유틸리티 — 빠르게 스타일링, CSS 파일 관리 줄이고 싶을 때, Tailwind 사용 시
실무에서는 혼용이 일반적 — 공통 컴포넌트는 BEM, 간단한 조정은 유틸리티
PART 11
Thymeleaf 기초 — 데이터 바인딩
Spring과 함께 쓰는 서버사이드 템플릿 엔진 — 서버 데이터를 HTML에 바인딩
기본 설정 — 네임스페이스 선언
<html xmlns:th="http://www.thymeleaf.org">
html 태그에 th 네임스페이스 선언 필수. 이후 th: 속성 사용 가능
th:text / th:value — 데이터 출력
<!-- 텍스트 출력 --> <p th:text="${user.name}">홍길동</p> <!-- input 기본값 --> <input th:value="${item.price}"> <!-- 링크 --> <a th:href="@{/detail/{id}(id=${item.id})}"> 상세보기 </a>
th:if / th:unless — 조건부 렌더링
<div th:if="${user.isAdmin}"> 관리자 메뉴 </div> <div th:unless="${user.isAdmin}"> 일반 사용자 메뉴 </div>
th:each — 반복 렌더링
<ul> <li th:each="item : ${items}" th:text="${item.name}"></li> </ul> <!-- 인덱스 포함 --> <tr th:each="item, stat : ${list}"> <td th:text="${stat.index + 1}"></td> <td th:text="${item.name}"></td> </tr>
th:classappend — 동적 클래스
<div class="card" th:classappend="${item.active} ? 'card--active' : ''"> </div> <span th:style="'color:' + ${item.color}"> </span>
💡 ${...} 변수  |  @{...} URL  |  #{...} 메시지(i18n)  |  *{...} 선택 변수
PART 12
Thymeleaf 레이아웃 · 컴포넌트
header · footer · 공통 UI를 한 곳에서 관리 — 수정 한 번으로 전 페이지 반영
th:fragment — 컴포넌트 정의
재사용할 HTML 조각을 fragment로 선언
<!-- fragments/header.html --> <header th:fragment="header"> <nav> <a th:href="@{/}">홈</a> <a th:href="@{/about}">소개</a> </nav> </header> <!-- fragments/footer.html --> <footer th:fragment="footer"> <p>© 2025 My Site</p> </footer>
th:replace / th:insert — 삽입
<body> <!-- replace: 태그째 교체 (권장) --> <header th:replace="~{fragments/header :: header}"> </header> <main>페이지 본문</main> <!-- insert: 태그 안에 삽입 --> <div th:insert="~{fragments/footer :: footer}"> </div> </body>
Layout Dialect — 레이아웃 상속
레이아웃 파일 하나를 만들고 페이지별 콘텐츠만 교체
<!-- layouts/default.html --> <html><body> <header th:replace="~{fragments/header :: header}"></header> <main layout:fragment="content"> <!-- 각 페이지가 이 부분을 채움 --> </main> <footer th:replace="~{fragments/footer :: footer}"></footer> </body></html> <!-- page/home.html (개별 페이지) --> <html layout:decorate="~{layouts/default}"> <main layout:fragment="content"> <h1>홈 페이지 내용</h1> </main></html>
replace vs insert 차이
th:replace — 해당 태그 자체를 fragment로 완전 교체. 태그 중복 없이 깔끔 ← 권장
th:insert — 해당 태그 안에 fragment 삽입. 감싸는 태그가 한 겹 더 생김
💡 header · footer · nav 등 공통 요소는 fragment로 분리 → 한 파일만 수정하면 전 페이지 반영
